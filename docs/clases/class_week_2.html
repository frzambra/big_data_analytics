Clase 2 - Big Data y Analytics

¿Qué Vimos la Semana Pasada?

  • Las empresas generan millones de datos cada día
  • Los datos por sí solos no son útiles
  • Transformamos datos → información → conocimiento → decisión
  • Los Sistemas de Información hacen posible esa transformación

Hoy: ¿De qué está hecho un Sistema de Información?

Repaso rápido. Preguntar si alguien recuerda el ejemplo de TechStyle. Conectar directamente con la clase anterior.

Situación: TechStyle Crece

El problema de éxito de TechStyle

TechStyle pasó de 50.000 a 450.000 clientes en 3 años.

El CEO le pregunta al equipo técnico:

“Nuestro sistema actual está colapsando. ¿Qué necesitamos para seguir creciendo?”

Presentar el escenario como continuación natural. El crecimiento trae nuevos desafíos tecnológicos y organizacionales.

El Equipo Técnico Responde

“Para escalar, necesitamos fortalecer los 5 componentes de nuestro SI”

┌─────────────────────────────────────────────┐
│           SISTEMA DE INFORMACIÓN            │
│                                             │
│  Hardware  +  Software  +  Datos            │
│                                             │

¿Qué significa cada uno?

Introducir los 5 componentes como respuesta a una necesidad real de negocio. No son conceptos abstractos: TechStyle los necesita para sobrevivir.

Componente 1: Hardware

La infraestructura física que procesa y almacena datos

  • Servidores: almacenan las bases de datos de clientes y pedidos
  • Dispositivos de red: conectan las tiendas, bodegas y oficinas
  • Terminales: computadores del equipo de marketing y logística
  • Dispositivos móviles: los repartidores registran las entregas

TechStyle tiene 12 servidores en la nube procesando 24/7

El hardware es tangible. Pedir a los estudiantes que piensen qué hardware usan ellos para interactuar con TechStyle (su celular al comprar).

Componente 2: Software

Los programas que procesan, analizan y presentan datos

Tipo Ejemplo en TechStyle Para qué sirve
Sistema operativo Linux Server Base del sistema
SGBD PostgreSQL Gestiona las BD
App web Tienda online Recibe pedidos
Analítico Power BI Genera reportes
ERP SAP Integra procesos

Sin software, el hardware es solo hierro

SGBD = Sistema Gestor de Base de Datos. Sera clave para el resto del curso. Mencionarlo brevemente.

Componente 3: Datos

El recurso más valioso de la organización

  • Datos de clientes: quiénes son, dónde están, qué prefieren
  • Datos de transacciones: qué compran, cuándo, a qué precio
  • Datos de productos: stock, costos, proveedores
  • Datos de logística: tiempos de entrega, rutas, incidentes

TechStyle tiene 8.750.000 registros de ventas → activo estratégico

Los datos son el “petróleo” del siglo XXI. Sin datos, el sistema no tiene nada que procesar ni analizar.

Componente 4: Personas

Los actores que operan, mantienen y usan el SI

┌─────────────────────────────────────────────────┐
│                   PERSONAS                      │
│                                                 │
│  Directivos  →  usan la información             │
│  Analistas   →  interpretan los datos           │
│  Técnicos    →  mantienen el sistema            │
│  Usuarios    →  ingresan y consultan datos      │
└─────────────────────────────────────────────────┘

El componente más crítico — y más complejo

Enfatizar que la tecnología sin personas no sirve. Los sistemas fracasan por problemas humanos, no técnicos.

Componente 5: Procesos

Las reglas y procedimientos que guían el uso del SI

  • ¿Cómo se registra un pedido?
  • ¿Quién puede acceder a qué información?
  • ¿Cómo se actualiza el inventario tras una venta?
  • ¿Cuándo se generan los reportes de ventas?
  • ¿Cómo se responde a un reclamo de cliente?

Sin procesos, el SI sería caótico aunque tuviera tecnología de punta

Los procesos son las “reglas del juego”. Poner ejemplo: si no hay proceso para actualizar stock, dos personas pueden vender el mismo producto que no existe.

Los 5 Componentes Interactúan

         [HARDWARE]
              ↕
         [SOFTWARE]
              ↕
           [DATOS]  ←→  [PROCESOS]
              ↕
          [PERSONAS]

Un SI es tan fuerte como su componente más débil

Analogía: como una cadena. Si falla un eslabón, falla toda la cadena. Ejemplo: excelente hardware, pero datos mal ingresados = información basura.

Actores y Roles en un SI

¿Quiénes son las personas en un SI?

No todos interactúan igual con el sistema.

Depende de: - Nivel jerárquico (operativo, táctico, estratégico) - Tipo de acceso (lectura, escritura, administración) - Propósito (ingresar datos, analizar, decidir)

Conectar con la idea de que las personas son uno de los 5 componentes. Ahora vamos a profundizar en ese componente.

El Modelo de Niveles Organizacionales

         ┌─────────────────────┐
         │   NIVEL ESTRATÉGICO │  → Alta gerencia
         │      (Decisión)     │
         ├─────────────────────┤
         │   NIVEL TÁCTICO     │  → Gerentes medios
         │    (Coordinación)   │
         ├─────────────────────┤
         │   NIVEL OPERATIVO   │  → Personal de campo
         │    (Ejecución)      │
         └─────────────────────┘

Cada nivel necesita distintos tipos de información

Este modelo es fundamental. Los distintos tipos de SI que veremos después corresponden a estos niveles. Plantar la semilla.

Nivel Operativo: Usuarios Finales

Los que ejecutan las operaciones del día a día

  • Cajeros / operadores de ventas: registran transacciones
  • Repartidores: confirman entregas en el sistema
  • Bodegueros: actualizan el inventario
  • Call center: ingresan reclamos y consultas

Necesitan: interfaces simples, rápidas, sin errores posibles

En TechStyle: cuando el repartidor marca “entregado” en la app, está ingresando un dato al SI. Es usuario operativo.

Nivel Táctico: Analistas y Gerentes Medios

Los que coordinan y toman decisiones intermedias

  • Gerente de marketing: ¿Qué campaña lanzar este mes?
  • Jefe de logística: ¿Cuántos repartidores necesito el sábado?
  • Analista de datos: ¿Qué productos están a punto de agotarse?
  • Supervisor de ventas: ¿Por qué cayeron las ventas en la zona norte?

Necesitan: reportes, tendencias, comparaciones, alertas

María González (la gerente de marketing del caso de la semana pasada) es un usuario táctico. Usó el SI para decidir dónde invertir los $15M.

Nivel Estratégico: Alta Dirección

Los que definen el rumbo de la organización

  • CEO: ¿Debemos expandirnos a nuevos mercados?
  • CFO: ¿La empresa es rentable en el largo plazo?
  • Directorio: ¿Cumplimos los objetivos estratégicos?

Necesitan: resúmenes ejecutivos, indicadores clave (KPIs), proyecciones

La alta dirección no quiere ver 8 millones de registros. Quiere ver: “Crecimos 15% este trimestre”. El SI debe entregar eso.

Tipos de Sistemas de Información

Un SI se diseña para servir a un nivel específico

  Nivel Estratégico  →  EIS  (Executive Information Systems)
  Nivel Táctico      →  DSS  (Decision Support Systems)
                     →  BI   (Business Intelligence)
                     →  CRM  (Customer Relationship Mgmt)
  Nivel Operativo    →  TPS  (Transaction Processing Systems)

No dar muchos detalles aún. Presentar el mapa general. Ahora vamos a ver cada uno con ejemplos de TechStyle.

TPS: Sistemas de Procesamiento de Transacciones

El motor del nivel operativo

  • Registran cada operación en tiempo real
  • Altamente confiables y rápidos
  • Manejan grandes volúmenes de transacciones
  • Ejemplos: registro de ventas, pagos, actualizaciones de inventario

En TechStyle: cada compra online es una transacción registrada automáticamente en la base de datos

TPS es la base de todo. Sin un buen TPS, los datos son incorrectos y todo lo que viene encima falla. Conectar con la importancia de la calidad de datos.

DSS: Sistemas de Apoyo a la Decisión

Herramientas para analizar situaciones complejas

  • Permiten hacer simulaciones: “¿Qué pasaría si…?”
  • Combinan datos internos y externos
  • No toman la decisión, la apoyan
  • Requieren capacidad analítica del usuario

En TechStyle: “Si bajo el precio de Electrónica en 10%, ¿cuánto aumentan las ventas?”

El DSS no dice “baja el precio”. Muestra escenarios posibles. La decisión la toma María, la gerente.

BI: Business Intelligence

Convirtiendo datos históricos en conocimiento estratégico

  • Analiza grandes volúmenes de datos históricos
  • Genera reportes, dashboards y visualizaciones
  • Detecta tendencias y patrones
  • Compara periodos: este mes vs. el año pasado

En TechStyle: el dashboard de ventas que María usa cada mañana muestra KPIs clave del negocio

BI es “mirar el retrovisor” con inteligencia. No predice el futuro, pero entiende el pasado muy bien. Es el tipo de sistema más usado en analítica de negocios.

CRM: Customer Relationship Management

Gestión de la relación con los clientes

  • Registra cada interacción con el cliente
  • Historial de compras, reclamos, preferencias
  • Permite personalizar la comunicación
  • Detecta clientes en riesgo de abandono (churn)

En TechStyle: “Pedro compró 3 veces en el último mes, pero lleva 60 días sin comprar → enviarle una oferta personalizada”

El CRM transforma a los clientes de números en personas con historia. Es crítico en e-commerce donde no hay interacción cara a cara.

EIS: Executive Information Systems

Información ejecutiva para decisiones estratégicas

  • Resúmenes de alto nivel (KPIs, indicadores)
  • Información en tiempo real, visual, sin tecnicismos
  • Alertas automáticas cuando algo sale mal
  • Diseñado para personas con poco tiempo y mucho poder decisional

En TechStyle: el CEO ve en su tablet: ventas del día, satisfacción de clientes, margen operacional

El EIS no muestra tablas con millones de filas. Muestra semáforos: verde, amarillo, rojo. Si algo está en rojo, el ejecutivo sabe que debe actuar.

Resumen: Los SI de TechStyle

 CEO Roberto    → EIS  → "Ventas del mes: $1.2B (+8% vs meta)"

 Gerente María  → DSS  → "¿Qué pasa si invierto en Hogar?"
                 → BI   → "Tendencias de ventas por categoría"

 Analista Juan  → BI   → "Reporte detallado por región"
                 → CRM → "Clientes con alto riesgo de churn"

 Repartidora Sofía → TPS → "Registrar entrega pedido #875432"

Reflexión: Más que Tecnología

¿Por qué no basta con comprar el mejor software?

  • El SI más caro falla si las personas no lo adoptan
  • Los mejores datos no sirven si los procesos son deficientes
  • El mejor hardware no ayuda si el software está mal configurado
  • La mejor tecnología no reemplaza la calidad de los datos

El SI es un sistema socio-técnico: personas + tecnología

Desmitificar la idea de que “con tecnología se resuelve todo”. El cambio organizacional es tan importante como la tecnología.

Conexión con la Toma de Decisiones

La cadena completa en TechStyle

  Datos de ventas (TPS)
        ↓
  Análisis de tendencias (BI)
        ↓
  Simulación de escenarios (DSS)
        ↓
  Dashboard ejecutivo (EIS)
        ↓
  Decisión: invertir $15M en categoría Hogar

Cada tipo de SI aporta un eslabón de la cadena decisional

Mostrar que los tipos de SI no son competencia entre sí: son complementarios. Se usan juntos para apoyar decisiones de distintos niveles.

Lo Que Aprendimos Hoy

  1. Un SI tiene 5 componentes: hardware, software, datos, personas, procesos
  2. Existen 3 niveles organizacionales con distintas necesidades de información
  3. Los actores interactúan con el SI según su rol y nivel
  4. Existen distintos tipos de SI: TPS, DSS, BI, CRM, EIS
  5. Cada tipo sirve a un propósito diferente en la toma de decisiones

Las organizaciones usan múltiples SI que trabajan juntos
