UPDATE: Modificar Datos

-- Actualizar el stock de un producto
UPDATE productos
SET stock = stock - 5
WHERE id_producto = 1;

-- Cambiar el estado de un pedido
UPDATE pedidos
SET estado = 'entregado'
WHERE id_pedido = 3;

-- Actualizar el email de un cliente
UPDATE clientes
SET email = 'ana.gonzalez@techstyle.cl'
WHERE id_cliente = 1;

¡Siempre usar WHERE! — Sin WHERE, se actualizan todos los registros de la tabla.

DELETE: Eliminar Datos

-- Eliminar una devolución específica
DELETE FROM devoluciones
WHERE id_devolucion = 5;

-- ¿Puedo eliminar un cliente que tiene pedidos?
DELETE FROM clientes WHERE id_cliente = 1;
-- ERROR 1451: Cannot delete or update a parent row:
-- a foreign key constraint fails

La FK protege contra eliminaciones accidentales.

Para eliminar Ana, primero debo eliminar (o reasignar) todos sus pedidos.

SELECT Básico: Consultar la Base de Datos

-- Todos los clientes
SELECT * FROM clientes;

-- Solo nombre y email
SELECT nombre, email FROM clientes;

-- Clientes de la Región Metropolitana
SELECT nombre, email FROM clientes
WHERE region = 'Metropolitana';

-- Productos con precio mayor a $20.000
SELECT nombre, categoria, precio FROM productos
WHERE precio > 20000
ORDER BY precio DESC;

-- Los 5 pedidos más recientes
SELECT id_pedido, fecha_pedido, total, estado
FROM pedidos
ORDER BY fecha_pedido DESC
LIMIT 5;

Operadores en WHERE

Operador Ejemplo
= WHERE estado = 'entregado'
<> o != WHERE estado <> 'cancelado'
>, <, >=, <= WHERE precio >= 20000
BETWEEN WHERE precio BETWEEN 10000 AND 50000
LIKE WHERE nombre LIKE '%Runner%'
IN WHERE categoria IN ('Ropa', 'Calzado')
IS NULL WHERE telefono IS NULL

Respondiendo Preguntas de Negocio con SELECT

Pregunta de la Gerente de Marketing:

¿Cuáles son los productos actualmente con stock bajo (menos de 50 unidades)?

SELECT nombre, categoria, precio, stock
FROM productos
WHERE stock < 50
ORDER BY stock ASC;

Pregunta del equipo de logística:

¿Qué pedidos están en estado ‘pendiente’ desde hace más de 3 días?

SELECT id_pedido, fecha_pedido, total
FROM pedidos
WHERE estado = 'pendiente'
  AND fecha_pedido < DATE_SUB(CURDATE(), INTERVAL 3 DAY);

Integridad al Insertar y Actualizar

  • No insertar datos falsos: Si el cliente dice que vive en “Santiago” pero la zona es de Valparaíso, el dato es incorrecto desde el origen.
  • Validar antes de actualizar: Cambiar el precio de un producto afecta todos los cálculos futuros del CMI.
  • Eliminar con cuidado: Un DELETE sin WHERE es irreversible.

Buena práctica: Antes de ejecutar UPDATE o DELETE, primero ejecutar el equivalente como SELECT para verificar qué registros serán afectados.

  • Cada consulta SELECT responde una pregunta de negocio concreta.

  • La consulta correcta es la diferencia entre una decisión informada y una decisión basada en suposiciones.
