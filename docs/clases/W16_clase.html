¿Qué es una Vista SQL?

  • Una vista es una consulta guardada que se comporta como una tabla virtual.
  • No almacena datos: cada vez que se consulta, ejecuta la consulta subyacente.
  • Permite encapsular consultas complejas y reutilizarlas con un nombre simple.
-- Crear una vista
CREATE VIEW nombre_vista AS
    SELECT ... FROM ... JOIN ... WHERE ...;

-- Consultar la vista (igual que una tabla)
SELECT * FROM nombre_vista;

-- Filtrar sobre la vista
SELECT * FROM nombre_vista WHERE mes = 6;

Ventajas de las Vistas

Beneficio Descripción
Simplicidad El analista de Power BI consulta vw_ventas_mes en vez de un JOIN complejo de 4 tablas
Seguridad Se puede dar acceso a la vista pero no a las tablas originales
Reutilización La misma consulta compleja usada por múltiples sistemas
Abstracción Cambios en la BD se reflejan automáticamente en la vista

Aplicación: ¿Por qué Vistas y no Consultas Guardadas?

Escenario: María en Power BI necesita el KPI de ventas por categoría.

  • Sin vista: Cada vez que María actualiza el dashboard, Power BI ejecuta el JOIN complejo de 4 tablas.
  • Con vista: Power BI consulta SELECT * FROM vw_kpi_ventas_categoria WHERE anio = 2026. Simple.
  • Ventaja de seguridad: María tiene acceso a vw_kpi_ventas_categoria, pero NO a las tablas pedidos, clientes ni order_items con datos sensibles.

Principio de mínimo privilegio aplicado al acceso a datos.

Vista 1: KPI — Ventas por Categoría (Perspectiva Financiera del CMI)

CREATE VIEW vw_kpi_ventas_categoria AS
SELECT
    pr.categoria,
    SUM(oi.cantidad * oi.precio_unitario) AS ventas_totales,
    COUNT(DISTINCT p.id_pedido)           AS num_pedidos,
    MONTH(p.fecha_pedido)                 AS mes,
    YEAR(p.fecha_pedido)                  AS anio
FROM order_items oi
INNER JOIN pedidos   p  ON oi.id_pedido   = p.id_pedido
INNER JOIN productos pr ON oi.id_producto = pr.id_producto
WHERE p.estado = 'entregado'
GROUP BY pr.categoria, YEAR(p.fecha_pedido), MONTH(p.fecha_pedido);

Consulta de uso: SELECT * FROM vw_kpi_ventas_categoria WHERE anio = 2026 AND mes = 6;

Vista 2: KPI — Ticket Promedio por Región (Perspectiva Cliente)

CREATE VIEW vw_kpi_ticket_por_region AS
SELECT
    c.region,
    COUNT(p.id_pedido)  AS total_pedidos,
    AVG(p.total)        AS ticket_promedio,
    SUM(p.total)        AS ingresos_totales
FROM pedidos p
INNER JOIN clientes c ON p.id_cliente = c.id_cliente
WHERE p.estado = 'entregado'
GROUP BY c.region
ORDER BY ingresos_totales DESC;

KPI del CMI alimentado: “Ticket promedio por región” → perspectiva Cliente del Balanced Scorecard.

Vista 3: KPI — Tasa de Devolución (Perspectiva Procesos)

CREATE VIEW vw_kpi_tasa_devolucion AS
SELECT
    COUNT(DISTINCT d.id_devolucion) AS total_devoluciones,
    COUNT(DISTINCT p.id_pedido)     AS total_pedidos,
    ROUND(
        COUNT(DISTINCT d.id_devolucion) * 100.0 /
        NULLIF(COUNT(DISTINCT p.id_pedido), 0), 2
    ) AS tasa_devolucion_pct
FROM pedidos p
LEFT JOIN devoluciones d ON p.id_pedido = d.id_pedido
WHERE p.estado IN ('entregado', 'cancelado');

NULLIF(..., 0): evita división por cero si no hay pedidos registrados.

Vista 4: KPI — Clientes Inactivos (Perspectiva Aprendizaje)

CREATE VIEW vw_kpi_clientes_inactivos AS
SELECT
    c.id_cliente,
    c.nombre,
    c.email,
    c.region,
    MAX(p.fecha_pedido) AS ultimo_pedido,
    DATEDIFF(CURDATE(), MAX(p.fecha_pedido)) AS dias_sin_comprar
FROM clientes c
LEFT JOIN pedidos p ON c.id_cliente = p.id_cliente
GROUP BY c.id_cliente, c.nombre, c.email, c.region
HAVING dias_sin_comprar > 60
    OR MAX(p.fecha_pedido) IS NULL
ORDER BY dias_sin_comprar DESC;

Caso de uso: María consulta esta vista para diseñar la campaña de reactivación de clientes inactivos.

Aplicación: Seguridad con Vistas

Problema: El equipo de marketing necesita acceso a datos de ventas para campañas, pero no debe ver datos sensibles de clientes.

-- Crear usuario de solo lectura para marketing
CREATE USER 'marketing_user'@'localhost' IDENTIFIED BY 'password123';

-- Dar acceso SOLO a las vistas del CMI, no a las tablas base
GRANT SELECT ON techstyle_db.vw_kpi_ventas_categoria TO 'marketing_user'@'localhost';
GRANT SELECT ON techstyle_db.vw_kpi_ticket_por_region TO 'marketing_user'@'localhost';
GRANT SELECT ON techstyle_db.vw_kpi_clientes_inactivos TO 'marketing_user'@'localhost';

El equipo de marketing ve los KPIs pero no puede consultar emails ni datos personales de clientes directamente.
