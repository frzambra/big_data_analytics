Laboratorio 3: Auditoría de Calidad y CMI con Datos Reales

Semana 3 · Big Data y Analytics · Universidad San Sebastián

1 Contexto

TechStyle detectó que su CMI muestra métricas inconsistentes. El CEO Roberto no confía en los datos del dashboard. El equipo sospecha que hay problemas de calidad en la tabla de clientes.

Tu tarea tiene dos partes:

  1. Auditar la tabla de clientes e identificar los problemas de calidad
  2. Construir el CMI de TechStyle en Power BI usando los datos de ventas

Al finalizar, descubrirás que un KPI del CMI no puede calcularse correctamente debido a los problemas de datos encontrados.

2 Instrucciones generales

  • Trabajarás individualmente.
  • Parte A: Power Query (auditoría de clientes)
  • Parte B: Power BI (construcción del CMI)
  • Duración total: 35 minutos

Archivos de datos:

  • labs/data/techstyle_customers_dirty.csv → Parte A
  • labs/data/techstyle_orders.csv → Parte B

3 Parte A — Auditoría de Calidad de Datos (18 min)

3.1 Paso 1: Cargar los datos de clientes

  1. Abre Power BI Desktop (o Excel con Power Query)
  2. Inicio → Obtener datos → Texto/CSV
  3. Carga techstyle_customers_dirty.csv
  4. Haz clic en Transformar datos (no en “Cargar”) — esto abre Power Query Editor

3.2 Paso 2: Exploración inicial

En Power Query Editor, usa las herramientas de perfilado de datos:

  1. Activa: Vista → Calidad de columna
  2. Activa: Vista → Distribución de columna
  3. Activa: Vista → Perfil de columna

Esto te mostrará automáticamente cuántos valores válidos, errores y vacíos tiene cada columna.

3.3 Paso 3: Completar la Tabla de Auditoría

Para cada columna, registra lo que encuentres:

Columna % Vacíos Problemas encontrados Dimensión de calidad Decisión de negocio afectada
email
apellido
region
fecha_nacimiento
fecha_registro
telefono

Dimensiones de calidad: Exactitud · Completitud · Consistencia · Oportunidad

Tip

Pista para emails inválidos: Usa Inicio → Filtrar filas → No contiene con el texto @ para encontrar emails sin arroba. Luego prueba con .com para encontrar dominios incompletos.

3.4 Paso 4: Identificar duplicados

  1. Selecciona la columna email
  2. Inicio → Quitar filas → Quitar duplicados
  3. Compara el número de filas antes y después.
    • Filas originales: ______
    • Filas tras eliminar duplicados de email: ______
    • Duplicados encontrados: ______

3.5 Paso 5: Reglas de validación

Para cada problema encontrado, propón una regla de validación que debería tener el sistema al momento de registrar un nuevo cliente:

Problema Regla de validación propuesta
Email inválido
Email duplicado
Región vacía
Fecha de nacimiento vacía

3.6 Paso 6: Reflexión de calidad

Responde:

  1. Si TechStyle quisiera hacer una campaña de email marketing a todos sus clientes, ¿qué porcentaje del total podría recibir el correo? (considera emails vacíos + inválidos)

  2. Si quisiera segmentar clientes por grupo etario (jóvenes, adultos, adultos mayores), ¿podría hacerlo con este dataset? ¿Por qué?

  3. ¿Qué área de la empresa es responsable de estos problemas? ¿Solo el área de TI?

4 Parte B — Construyendo el CMI en Power BI (17 min)

4.1 Paso 7: Cargar los datos de ventas

  1. En Power BI, carga también techstyle_orders.csv
  2. Ahora tienes dos tablas: techstyle_customers_dirty y techstyle_orders

4.2 Paso 8: Crear medidas para los KPIs del CMI

Crea las siguientes medidas DAX (botón derecho sobre la tabla → Nueva medida):

Perspectiva Financiera:

Total Ventas = SUM(techstyle_orders[venta_neta])
Margen Promedio = AVERAGE(techstyle_orders[margen_pct])

Perspectiva Cliente:

Total Clientes con Email =
COUNTROWS(
    FILTER(techstyle_customers_dirty,
           techstyle_customers_dirty[email] <> "")
)

Perspectiva Procesos:

Pedidos por Canal Web =
CALCULATE(
    DISTINCTCOUNT(techstyle_orders[order_id]),
    techstyle_orders[canal_venta] = "Web"
)

4.3 Paso 9: Construir las 4 perspectivas del CMI

Crea una página llamada "CMI TechStyle" con esta estructura:

Fila 1 — Perspectiva Financiera (color verde):

  • Tarjeta: Total Ventas
  • Tarjeta: Margen Promedio
  • Meta referencial: ventas > $50.000.000 CLP / margen > 25%

Fila 2 — Perspectiva Cliente (color azul):

  • Tarjeta: Total Clientes registrados (COUNT de id)
  • Tarjeta: Clientes con email válido
  • Gráfico de barras: clientes por region

Fila 3 — Perspectiva Procesos (color naranja):

  • Gráfico de dona: ventas por canal_venta
  • Tabla: ventas por categoria con margen

Fila 4 — Perspectiva Aprendizaje (color morado):

  • Tarjeta con texto: “KPI: % clientes con perfil completo”
  • Nota: este KPI no puede calcularse correctamente (¿por qué?)

4.4 Paso 10: El KPI que falla

Important

El KPI de Aprendizaje que Roberto quería medir era: “¿Qué porcentaje de nuestros clientes tiene perfil completo (con edad, región y email válido)?”

Este indicador es clave porque permite medir la calidad del registro de clientes como proceso interno.

Pero no puedes calcularlo correctamente. ¿Por qué?

Responde:

  1. ¿Qué datos te faltan o están mal para calcular este KPI?
  2. ¿Qué decisiones de negocio no puede tomar Roberto porque este KPI está dañado?
  3. ¿Qué proceso operativo deberían cambiar para que este KPI sea confiable en el futuro?

5 Reflexión final

Conecta ambas partes:

El CMI de TechStyle es una herramienta estratégica poderosa. Pero su confiabilidad depende completamente de la calidad de los datos que lo alimentan.

Completa la frase:

“Si TechStyle no corrige los problemas de calidad en la tabla de clientes, el CEO Roberto tomará decisiones incorrectas sobre _______ porque el dato de _______ está _______ (vacío / incorrecto / duplicado).”

6 Entrega

Guarda el archivo Power BI como lab_semana3_[apellido].pbix.

El docente preguntará: “¿Quién encontró el mayor número de problemas de calidad? ¿Cuántos eran?”
